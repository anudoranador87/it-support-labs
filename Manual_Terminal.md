# 📖 Survival Manual: Linux Terminal for IT Support

This manual contains essential commands that every support technician should know to manage Linux systems.

## 1. Basic Navigation
* `pwd`: Where am I? (Print Working Directory).
* `ls -lah`: List files (including hidden ones with human-readable sizes).
* `cd <folder>`: Enter a folder.
* `cd ..`: Go up one level.

## 2. File Management
* `cp <source> <destination>`: Copy files.
* `mv <source> <destination>`: Move or rename files.
* `rm <file>`: Delete file. (**Be careful!**).
* `mkdir <name>`: Create a new folder.

## 3. Diagnostics and Logs
* `tail -f /var/log/syslog`: View errors in real-time.
* `grep "error" <file>`: Search for a specific word.
* `df -h`: View free disk space.
* `top`: View processes and CPU/RAM consumption.

## 4. Networking
* `ip a`: View my IP address.
* `ping <ip/web>`: Check connectivity.
* `ss -tuln`: View open ports.

## 5. Permissions and Administration
* `sudo <command>`: Run as administrator.
* `chmod +x <file>`: Make a file executable (needed for scripts).
* `systemctl status <service>`: Check if a program is running.
